# Регламент деплоя

## Окна деплоя

Плановые деплои продакшена выполняются **по вторникам и четвергам
с 14:00 до 17:00 мск** — в это время дежурит полный состав SRE.
Деплой в пятницу после 15:00 и в выходные запрещён (кроме хотфиксов
уровня P1 с одобрения дежурного SRE-лида).

## Порядок

1. Изменение влито в `main`, CI зелёный, фича-флаг для нового поведения
   выключен по умолчанию.
2. Деплой запускается из деплой-бота командой:

```bash
/deploy service=billing env=staging ref=main
# после проверки на staging:
/deploy service=billing env=prod ref=main
```

3. Канареечная стадия: новая версия получает 5% трафика на 30 минут;
   бот следит за error rate и p95. Превышение порогов — автооткат.
4. Полная раскатка подтверждается инженером в треде деплоя.

## Откат

Откат — той же командой с указанием предыдущего релиза:

```bash
/deploy service=billing env=prod ref=release-2026.07.10 --rollback
```

Откат не требует аппрувов и доступен любому инженеру команды сервиса
в любое время.

## Миграции БД

Миграции выполняются отдельным шагом до деплоя кода и обязаны быть
обратно совместимыми на один релиз (expand-contract). Деструктивные
миграции — только через 2 релиза после расширяющей.
