## Рамки фаз

### P1 — Доступ уровня предприятия (2–3 недели)
OIDC/SSO (authlib, провайдер конфигурируемый); enforcement doc-level ACL: наследование прав Confluence-space → `chunks.metadata.acl` при ingest, `access_context` из JWT прокидывается в оба канала retrieval **до ранжирования** (`docs/security-and-access.md` §3 — включая nearest_documents и /search); NER-PII при ingest (presidio, политики per collection: block/mask/tag); append-only аудит. **Ключевые тесты:** «утечки через все пути выдачи» — параметризованный тест: пользователь без прав не видит документ ни в answer, ни в citations, ни в nearest_documents, ни в /search, ни в кэшированных ответах.

### P2 — Мультитенантность и новые источники (3–4 недели)
PostgreSQL RLS по tenant_id (страховочный слой поверх репозиториев); tenant в JWT; **tenant-scoped ключи кэша Redis** (критично — см. security §4); онбординг-flow tenant; коннекторы Notion и Google Drive по контракту `SourceConnector` (`docs/adr/ADR-010`) — без изменений пайплайна; near-duplicate дедуп корпуса (minhash). **Ключевые тесты:** изоляция двух tenant на одном стенде, включая кэш и eval-данные.

### P3 — Масштаб: Qdrant, K8s, автоскейл (4–6 недель)
**Сначала новые ADR:** ADR-011 план миграции vector store (двойная запись → сверка → переключение → декомиссия; критерии отката), ADR-012 топология K8s (пулы: api / workers-по-очередям / GPU-инференс; HPA-сигналы: длина очередей Celery, latency инференса). Затем: `QdrantVectorStore` за существующим интерфейсом (`docs/adr/ADR-001` — интерфейс уже готов), backfill-джоб, Helm-чарты, managed PG + Redis Sentinel, облачный LLM как вторая реализация LLMClient (`docs/adr/ADR-009`), SLO-алертинг, нагрузочные до NFR-prod (`docs/nfr.md`). **Ворота:** eval-метрики не деградировали после миграции хранилища (полный run до/после).

### P4 — Граф-RAG и агентный поиск (4–8 недель, исследовательская)
**Сначала ADR:** ADR-013 граф знаний (извлечение сущностей/связей при ingest, хранилище, гранулярность), ADR-014 агентная топология (LangGraph-агент с MCP-инструментами: индекс + живые Confluence/Notion через MCP-серверы из ADR-010; бюджет шагов, деградация до обычного RAG). Плюс: learned sparse (bge-m3) как третий канал RRF; online-eval (canary-промпты, автосбор датасета из 👎). **Ворота:** каждый новый канал/агентный режим включается только при статистически значимом приросте на golden dataset (A/B через eval-runs), иначе остаётся за фиче-флагом.

## Сквозные правила трека
- Ни одна фаза не ломает `/api/v1` (совместимость — `docs/api-contract.md` §8).
- Каждая фаза заканчивается полным eval-run против baseline.
- Все новые решения — через ADR до кода.
