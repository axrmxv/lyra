# Промпт фазы 7 — Демо-упаковка и hardening

> Скопируй всё ниже в Claude Code. Предусловие: фазы 0–6 завершены (evals зелёные).

---

Проект **LYRA** — RAG-платформа корпоративных знаний. Финальная MVP-фаза: продукт должен запускаться с нуля за ≤15 минут и убедительно показываться на собеседовании. Прочитай перед началом: `docs/nfr.md` (целевые цифры — замеряем фактические), `docs/PRD.md` (все UC — прогоняем), `docs/security-and-access.md` §7, `PLAN.md` (фаза 7).

## Что реализовать

1. **`make seed-demo`**: одна команда — сидит demo-корпус (`evals/corpus/` из фазы 6 + 3–5 «витринных» документов с эффектными таблицами), пользователей (admin/editor/viewer с паролями из .env), запускает ingest и ждёт завершения с прогрессом.
2. **Смоук-прогон всех UC** (`scripts/demo_smoke.py`): автоматический сценарий UC-1..UC-9 через API с assert'ами; выводит таблицу PASS/FAIL — это и приёмка, и регресс-проверка перед каждым показом.
3. **Нагрузочный смоук** (`scripts/load_smoke.py` или locust-конфиг): (а) /search 20 RPS 2 мин; (б) 3 параллельных chat-диалога; (в) ingest 20 документов пачкой. Фактические p50/p95 против `docs/nfr.md` §1–2 → таблица в `docs/nfr-actual.md` (создать; расхождения честно фиксируются с комментарием).
4. **Security-гигиена** (чек по `docs/security-and-access.md`): CORS только фронтенд-origin; security-заголовки; все контейнеры non-root; pip-audit + npm audit job в CI (warning-режим, не гейт); grep-тест «нет секретов в репо» (gitleaks в pre-commit и CI); маскирование чувствительных полей в логах — проверить тестом.
5. **Отказоустойчивость руками** (сценарии из `docs/architecture.md` §4, зафиксировать результат в отчёте): stop reranker → degraded-ответы; stop ollama → 503 с внятной ошибкой в UI; stop redis → чат работает (без кэша), upload отдаёт 503; kill worker mid-job → job довыполняется без дублей.
6. **README.md** (витрина проекта): бейджи CI; 3 скриншота (чат с цитатами, отказ, Grafana); Mermaid-диаграмма архитектуры (сокращённая из `docs/architecture.md`); Quick start (3 команды); таблица фактических метрик качества (из последнего eval-run) и latency (из nfr-actual); ссылки на docs/ и ADR-индекс.
7. **`docs/demo-script.md`**: сценарий показа на 10 минут: (1) вопрос с ответом и цитатами → клик по сноске; (2) вопрос вне корпуса → честный отказ; (3) «кривая» формулировка → показать corrective-итерацию в trace_id логах; (4) загрузка нового документа → вопрос по нему; (5) eval-отчёт и красный CI при порче промпта; (6) Grafana-разбивка. К каждому шагу — конкретный вопрос-реплика и ожидаемый результат.

## Критерии приёмки
- Чистая машина (или `docker compose down -v`): `make up && make seed-demo` → рабочее демо ≤ 15 мин (замерить).
- `scripts/demo_smoke.py` — все UC PASS.
- Все eval-пороги зелёные на финальном коде; `docs/nfr-actual.md` заполнен.
- gitleaks чистый; аудиты зависимостей без критичных находок (или зафиксированы исключения с обоснованием).

## Тесты
- Новых юнит-тестов минимум: gitleaks/CI-джобы, тест маскирования логов, smoke-скрипты (они и есть приёмка).
- Полный `make test && make lint && make eval` — зелёный, это ворота фазы.
