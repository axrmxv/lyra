# ADR-007: Стратегия цитирования

**Статус:** принято · **Дата:** 2026-07-08

## Контекст

Ключевая ценность продукта — проверяемость: «ответам без источников нельзя доверять» ([PRD](../PRD.md), FR-10). Нужно решить, с какой гранулярностью цитировать, как заставить LLM цитировать корректно и как передавать цитаты в API. Ограничение: локальная 7B-модель ([ADR-009](ADR-009-llm-provider-abstraction.md)) хуже следует сложным форматам, чем frontier-модели — схема должна быть простой и валидируемой.

## Варианты

| Критерий | Inline-маркеры [n] на уровне предложений/абзацев (выбран) | Цитаты только списком в конце ответа | Span-level (точные символьные смещения в источнике) |
|----------|------------------------------------------------------------|--------------------------------------|------------------------------------------------------|
| Проверяемость читателем | ✅ Видно, какое утверждение на чём основано | ❌ Непонятно, что из чего следует | ✅ Максимальная |
| Посильность для 7B-модели | ✅ Простой формат, легко валидировать | ✅ | ❌ Модель ненадёжно выдаёт смещения |
| Автовалидация | ✅ Маркер → chunk_id, NLI/LLM-проверка поддержки | ⚠️ Только на уровне всего ответа | ✅, но хрупко |
| UX в чате | ✅ Кликабельные сноски | ⚠️ | ✅ Подсветка в источнике |
| Когда применять | Базовая стратегия production-RAG | Черновые прототипы | Frontier-модель + требования комплаенса |

## Решение

**Inline-маркеры `[n]` + структурированный массив citations**, с валидацией на узле `cite` и enforcement на узле `self_check` ([ADR-006](ADR-006-langgraph-topology.md)).

1. **Промпт generate:** контекст передаётся как нумерованные источники `[1]..[k]` (номер = позиция chunk после rerank); инструкция — каждое фактологическое утверждение завершать маркерами использованных источников; утверждениям без опоры в контексте — не быть.
2. **Узел cite** детерминированно (без LLM): парсит маркеры из текста, маппит на chunk_id, строит массив `citations`; правила валидации:
   - маркер, не соответствующий переданному источнику → ошибка формата → регенерация;
   - источник передан, но не процитирован ни разу при фактологическом ответе нулевой длины citations → подозрение на unsupported-ответ → self_check решает;
   - из процитированного chunk извлекается `quote` — наиболее релевантное предложение (совпадение по перекрытию с ответом) для отображения в UI.
3. **Узел self_check** (LLM-judge): для каждого предложения с маркером — подтверждает ли указанный источник утверждение; unsupported claims → регенерация (≤1) → честный отказ.
4. **Формат в API** ([api-contract.md](../api-contract.md)): `citations[] = {id, chunk_id, document_id, document_title, url, quote, relevance_score}`; в тексте ответа остаются маркеры `[n]`, фронтенд рендерит их сносками с popover и переходом к источнику.
5. **Confidence** (enum `high|medium|low` + score 0..1): взвешенная комбинация sufficiency.score, среднего rerank-score процитированных chunks, доли подтверждённых утверждений из self_check. Формула — конфиг, калибруется на golden dataset ([eval-plan.md](../eval-plan.md)).
6. Метрика **citation validity ≥ 0.95** — гейт в CI ([PRD](../PRD.md) §6).

## Последствия

- (+) Цитаты проверяются автоматически (детерминированная валидация + LLM-judge), а не «на честном слове» модели.
- (+) Формат посилен локальной модели; деградация формата ловится узлом cite до показа пользователю.
- (−) +1 LLM-вызов (self_check) на каждый ответ — заложено в бюджет латентности [ADR-006](ADR-006-langgraph-topology.md).
- (−) Гранулярность — chunk, не точный span: quote-выдержка эвристическая. Span-level отмечен как production-эксперимент при переходе на более сильную модель.
