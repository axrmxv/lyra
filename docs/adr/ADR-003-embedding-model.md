# ADR-003: Модель embeddings

**Статус:** принято · **Дата:** 2026-07-08

## Контекст

Нужна embedding-модель для векторизации chunks и запросов. Жёсткие требования: **сильный русский язык** (допущение A-1 в [PRD](../PRD.md)), self-hosted OSS (A-3), работа на CPU ноутбука с приемлемой скоростью ingest, размерность, разумная для pgvector HNSW. Желательно: длинный входной контекст (chunks до 768 токенов + контекстные заголовки), пригодность к гибридному поиску.

## Варианты

| Критерий | BAAI/bge-m3 | intfloat/multilingual-e5-large | OpenAI text-embedding-3-large (для сравнения) |
|----------|-------------|--------------------------------|-----------------------------------------------|
| Русский язык | ✅ Топ среди OSS на MIRACL/ruMTEB-классах задач | ✅ Хорош, чуть слабее bge-m3 на retrieval | ✅ Хорош |
| Self-hosted | ✅ | ✅ | ❌ Только API, платно, данные уходят наружу |
| Макс. вход | ✅ 8192 токена | ❌ 512 токенов — впритык к нашему chunk 512–768 | 8191 |
| Размерность | 1024 | 1024 | 3072 (или усечение) |
| Режимы | ✅ dense + sparse + ColBERT (multi-vector) — задел для learned sparse retrieval | dense | dense |
| Особенности использования | Инструкции не нужны | ⚠️ Требует префиксы "query:"/"passage:" — источник тихих багов | — |
| Размер модели / скорость CPU | ~2.2 GB, средняя | ~2.2 GB, сопоставимо | — |
| Инференс-сервинг | ✅ Поддержана в HF text-embeddings-inference (TEI) | ✅ TEI | — |
| Когда применять | Мультиязычный RAG self-hosted (наш случай) | Альтернатива, если bge-m3 не влезает по ресурсам | Когда допустим облачный API и бюджет |

## Решение

**BAAI/bge-m3** (dense-режим, 1024-мерные векторы, cosine), сервинг отдельным контейнером **text-embeddings-inference (TEI)** с HTTP API.

- Отдельный сервис (а не библиотека в процессе API/worker): одна загрузка модели в память, независимое масштабирование, одинаковый путь для ingest и query.
- Клиент — тонкий `EmbeddingClient` с батчингом, retry и rate limit (используется и ingest-пайплайном, и retrieval-слоем).
- Sparse/ColBERT-режимы bge-m3 в MVP не используются (BM25 покрывает лексический канал, [ADR-005](ADR-005-hybrid-search-fusion.md)); отмечены как production-эксперимент.
- Кэш эмбеддингов запросов в Redis (ключ — хэш нормализованного текста запроса).

**Критично:** модель эмбеддингов — часть контракта индекса. Смена модели = полная переиндексация + прогон evals. Имя и версия модели хранятся в метаданных коллекции ([data-model.md](../data-model.md)).

## Последствия

- (+) Русский язык покрыт лучшей доступной OSS-моделью; $0 за API; длинный вход снимает риск обрезки chunks.
- (+) Путь апгрейда (например, на облачные embeddings в production) изолирован `EmbeddingClient` + процедурой переиндексации.
- (−) CPU-инференс медленный: ingest больших корпусов — минуты-десятки минут; лечится батчингом и тем, что ingest асинхронный.
- (−) 1024 float32 ≈ 4 КБ на вектор; 50k chunks ≈ 200 МБ + HNSW — приемлемо для MVP, на росте — аргумент за квантизацию в Qdrant ([ADR-001](ADR-001-vector-store-pgvector-vs-qdrant.md)).
