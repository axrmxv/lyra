# Нефункциональные требования (NFR) — LYRA

Два столбца целей: **MVP** — честные цифры для локального стека (Ollama на CPU ноутбука, допущения A-2, A-10 [PRD](PRD.md)) и **Production target** — облачный LLM / GPU-инференс. NFR проверяются: latency — метриками Prometheus (гистограммы по этапам), нагрузочный смоук — в фазе 7 ([PLAN.md](../PLAN.md)).

---

## 1. Latency

| Операция | MVP p50 | MVP p95 | Prod p50 | Prod p95 | Комментарий |
|----------|---------|---------|----------|----------|-------------|
| **Chat: полный ответ** (happy path, без corrective) | ≤ 8 с | ≤ 20 с | ≤ 2.5 с | ≤ 6 с | Доминирует LLM-инференс на CPU; SSE-стриминг скрывает часть ожидания |
| Chat: **время до первого токена** | ≤ 4 с | ≤ 10 с | ≤ 1.2 с | ≤ 2.5 с | UX-критичная метрика: retrieval + grading + начало генерации |
| Chat: worst case (2 corrective + self-check retry) | ≤ 25 с | ≤ 60 с | ≤ 8 с | ≤ 15 с | Ограничен жёсткими лимитами циклов ([ADR-006](adr/ADR-006-langgraph-topology.md)); дольше 60 с — таймаут и честная ошибка |
| **/search** (raw retrieval, с rerank) | ≤ 700 мс | ≤ 2 с | ≤ 300 мс | ≤ 500 мс | Доминирует reranker на CPU (50 пар) |
| /search (без rerank / rerank degraded) | ≤ 200 мс | ≤ 500 мс | ≤ 100 мс | ≤ 250 мс | BM25 + HNSW + RRF |
| Upload → 202 (постановка job) | ≤ 200 мс | ≤ 500 мс | — | — | Синхронная часть ingest минимальна (FR-2) |

Разбивка бюджета chat p50 (MVP, CPU): embedding запроса ~50 мс (кэш → 0) + гибридный поиск ~150 мс + rerank ~1 с + grade_sufficiency ~1 с + генерация ~4–5 с (~200 токенов) + cite ~0 + self_check ~1.5 с.

## 2. Пропускная способность

| Показатель | MVP | Prod target |
|------------|-----|-------------|
| Одновременные chat-запросы | 2–3 (локальная LLM — узкое место, очередь на Ollama) | 50+ RPS на кластер (горизонтальный скейл инференса) |
| /search RPS | 10–20 | 200+ |
| Ingest | ≥ 10 документов/мин (средний документ ~10 страниц; embed на CPU — узкое место) | 100+ док/мин (GPU-эмбеддинг, пул воркеров) |
| Первичная синхронизация Confluence 1000 страниц | ≤ 2 ч | ≤ 15 мин |
| Размер корпуса | 500 документов / 50k chunks | 1M+ документов (после миграции на Qdrant, [ADR-001](adr/ADR-001-vector-store-pgvector-vs-qdrant.md)) |

Деградация под нагрузкой: очередь к Ollama > N → 429 с Retry-After (лучше честный отказ, чем таймауты у всех).

## 3. Стоимость на запрос

**MVP: $0 API.** Всё self-hosted ([ADR-009](adr/ADR-009-llm-provider-abstraction.md), A-3). Стоимость = амортизация железа + электричество; для портфолио фиксируем ресурсную цену:

| Ресурс | Оценка на chat-запрос (MVP) |
|--------|------------------------------|
| LLM-токены (все узлы, happy path) | ~7–9k prompt + ~0.7k completion (~4 вызова) |
| LLM-токены worst case | ~18–22k prompt (corrective ×2, retry) |
| CPU-время | ~30–60 с CPU-секунд (LLM+rerank) |
| RAM стека | ~12–16 ГБ (Ollama 7B q4 ~5 ГБ, embeddings ~2, reranker ~2, PG/Redis/API/UI ~3) |

**Production target (для сравнения, облачный LLM класса Claude Haiku):** happy path ~8k input + 0.7k output ≈ **$0.01–0.015/запрос** токены; бюджет ≤ **$0.03/запрос** полной стоимости (токены + инфраструктура) при 10k запросов/мес. Кэш retrieval и короткие grading-промпты — главные рычаги экономии. Метрика `usage` в каждом ответе ([api-contract.md](api-contract.md)) делает стоимость наблюдаемой с первого дня.

## 4. Доступность и надёжность

| Показатель | MVP | Prod target |
|------------|-----|-------------|
| Доступность | Best-effort (демо-стенд, один хост) | 99.5% (K8s, реплики API/воркеров/инференса) |
| RTO / RPO | Перезапуск compose ≤ 5 мин / бэкап PG ежедневный (RPO 24 ч) | RTO ≤ 15 мин / RPO ≤ 1 ч (PITR) |
| Потеря ingest-задач | Не допускается (acks_late + идемпотентность, [ADR-008](adr/ADR-008-task-queue-celery-vs-rq.md)) | То же + DLQ и алерты |
| Деградации | Определены per-компонент ([architecture.md §4](architecture.md)) | + автофейловер |

## 5. Ресурсные требования стенда MVP

Минимум: 4 CPU / 16 ГБ RAM / 20 ГБ диска (модели ~8 ГБ). Рекомендуется 8 CPU; GPU (8+ ГБ VRAM) снижает chat p50 до ~2–3 с без изменения кода (конфиг Ollama/TEI).

## 6. Прочие NFR

- **Наблюдаемость:** 100% LLM-вызовов трейсятся (FR-19); Prometheus-гистограммы latency по этапам пайплайна; Grafana-дашборд из репозитория (provisioning).
- **Совместимость:** API аддитивно-эволюционирует в /api/v1 ([api-contract.md §8](api-contract.md)).
- **Локализация:** UI и ответы — русский; корпус ru+en (A-1).
- **Запуск демо:** полный стек — один `docker compose up` + `make seed-demo` (корпус + пользователи) ≤ 15 мин на чистой машине, без внешних ключей.
