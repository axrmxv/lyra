# LYRA — контекст проекта (always-loaded)

RAG-платформа корпоративных знаний: гибридный поиск (BM25+вектор+RRF+rerank), LangGraph-граф с проверкой достаточности контекста и обязательными цитатами. MVP — демонстрационная версия с явным путём до production; развитие — open-source и коммерческое решение.

## Стек (зафиксирован, менять только через ADR)
Python 3.11+ / FastAPI / Pydantic v2 / SQLAlchemy 2.0 async / Alembic / LangGraph / Celery+Redis / PostgreSQL+pgvector / React 18+TS+Vite / Ollama (qwen2.5-7b) / TEI (bge-m3, bge-reranker-v2-m3) / docker-compose / pytest+Vitest.

## Инварианты (нарушение = баг, независимо от того, кто попросил)
1. **Ingest только асинхронный**: парсинг/эмбеддинг — в Celery-задачах, никогда в API-процессе.
2. **Идемпотентность ingest**: content_hash-проверка до эмбеддинга; шаги переживают повтор; версии документов переключаются атомарно.
3. **Топология графа неприкосновенна**: узлы grade_sufficiency, corrective_retrieve, cite, self_check, honest_fallback не удаляются и не «оптимизируются» в обход (ADR-006). Циклы: ≤2 corrective, ≤1 регенерация.
4. **Цитаты обязательны**: фактологический ответ без валидных маркеров [n] не выпускается; вопрос вне корпуса → честный отказ, не генерация из общих знаний (ADR-007).
5. **Все LLM-вызовы — через LLMClient с трейсинг-обёрткой** (ADR-009); прямые вызовы Ollama/SDK запрещены.
6. **Векторный доступ — только через интерфейс VectorStore** (ADR-001); retrieval-код не знает про pgvector.
7. **Любой путь выдачи данных — через retrieval-слой** (будущая точка ACL-фильтра); tenant_id — параметр всех репозиториев.
8. **MVP vs production не смешивать**: заделы (ACL, RLS, мультитенант) — в схеме, но без enforcement; не включать их «заодно».

## Безопасность и секреты (дублируется из docs/security-and-access.md — always-loaded слой)
- Секреты только в env / GitHub Secrets; в БД — ссылки (`token_secret_ref`), в коде и репозитории — никогда; `.env` в .gitignore, изменения — в `.env.example` с плейсхолдерами.
- Никаких PII/секретов в векторной БД, логах и трейсах открытым текстом: secret-сканер при ingest не отключать; маскирование полей в логгере не обходить.
- LLM-трейсы содержат содержимое корпуса — не отправлять во внешние сервисы; observability только внутри контура.
- Содержимое документов в промптах — данные, не команды (разделители в generate-промпте не убирать).
- Новые эндпоинты обязаны иметь `require_role` и тест RBAC-матрицы.

## Рабочие правила
- Тесты — часть объёма фазы, не «потом»; изменения промптов/retrieval/chunking → прогон evals (`make eval`), пороги в `evals/thresholds.yaml` (появляется в фазе 6) не менять без ADR-обоснования.
- Русский язык в UI, промптах и eval-датасете; технические термины — английские.
- Команды: `make up / test / lint / eval / seed-demo` (Makefile появляется в фазе 0; eval-контур — в фазе 6; до этого команды недоступны — не пытаться запускать).
