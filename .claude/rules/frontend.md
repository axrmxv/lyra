# Правила: frontend (`frontend/`)

- Типы API — вручную синхронизированы с `docs/api-contract.md` в `src/api/types.ts`; изменил контракт — обнови типы и наоборот.
- SSE-поток чата обрабатывается редьюсером (status/token/final/error) — новые события добавлять в редьюсер с тестом, не ad-hoc в компоненте.
- Маркеры `[n]` в ответе рендерятся сносками; маркер без соответствующей citation — не падать, рендерить как текст (деградация формата уже отловлена бэкендом, фронт — последняя линия).
- Refusal-ответ (`refusal: true`) — отдельное визуальное состояние с nearest_documents; не маскировать под обычный ответ.
- `confidence` и `degraded` всегда отображаются — это продуктовая честность, не украшение.
- UI-тексты — русский; ключи/код — английский. Токен — в памяти; в localStorage не класть.
- Vitest на: редьюсер SSE, рендер цитат, refusal-состояние — минимальный обязательный набор при изменениях чата.
