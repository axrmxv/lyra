# Правила: evals (`lyra/evals/`, `evals/`)

- Пороги в `evals/thresholds.yaml` НЕ снижать для «зелёного CI» — падение метрики это сигнал о регрессии кода/промптов, а не о строгих порогах. Изменение порога — отдельный PR с обоснованием в описании и ссылкой на ADR/решение.
- Golden dataset — append-only: item'ы не редактируются и не удаляются задним числом (ломает сравнимость run'ов); исправление = деактивация старого item + новый (см. `docs/eval-plan.md` §4).
- Синтетические item'ы без ручного ревью (`reviewed: true`) в golden set не попадают.
- Каждый run обязан писать `config_snapshot` (git_ref, версии промптов, модели, retrieval-параметры, chunking_config) — run без snapshot бесполезен для регресс-анализа.
- Judge-модель в runtime-коде не используется (только evals/CI); смена judge-модели = пересчёт baseline, старые числа несравнимы — фиксировать в отчёте run.
- temperature=0 во всех eval-прогонах (генерация и judge).
- Метрики считать по формулам `docs/eval-plan.md` §1; context_recall и валидация маркеров — детерминированные, LLM-judge для них не подменять.
- Демо-корпус `evals/corpus/` — фикстура: менять только вместе с пересмотром датасета и baseline.
